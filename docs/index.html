Local-first capture protocol

Portable bundles for posts, articles, media, and OCR.

Clipsmith separates deterministic capture protocol from platform browser strategy. Agents handle the messy web; the CLI validates and moves clean bundles.

View on GitHub Read the workflow

Why it exists

Capture is not knowledge management.

Small contract

Every capture becomes a directory with capture.json, post.md, summary.md, warnings, and status. OCR captures may also keep the source picture.

Agent execution

Platform skills keep CDP ports, proxies, login sessions, and fallback steps close to the capture workflow.

Downstream neutral

Clipsmith validates and exports bundles. Other tools decide how to review, archive, or turn them into knowledge.

Workflow

One protocol, different capture strategies.

Use the installed agent skill for real captures. Use the CLI for job state, validation, and sinks.

  1. 01 Start
    Select a provider and create a pending capture job.
  2. 02 Capture
    Run the provider skill for XHS, X, WeChat, web, or OCR.
  3. 03 Validate
    Check paths, required files, schema, status, and warnings.
  4. 04 Sink
    Copy the validated bundle only when explicitly requested.

Commands

The protocol surface is scriptable.

clipsmith providers --json
clipsmith capture start "https://example.com/post" --state-dir /tmp/clipsmith-state
clipsmith validate-bundle /path/to/bundle --json
clipsmith capture finalize "<job_id_or_job_path>" /path/to/bundle --state-dir /tmp/clipsmith-state
clipsmith sink inbox /path/to/bundle /path/to/inbox-workspace --json

Bundle

Portable by default.

  • Schema: clipsmith.capture_bundle.v1
  • Relative content and asset paths
  • Status and warnings preserved
  • Validation before reporting ready

Install

CLI plus skills.

uv tool install --force git+https://github.com/OctopusGarage/clipsmith.git
clipsmith install --all
clipsmith doctor
